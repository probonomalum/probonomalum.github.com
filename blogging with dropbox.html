Getting started

Dropbox as blogging platform

L'idea di base è di utilizzare una directory pubblica di dropbox come database per una piattaforma di blogging. ora sto modificando il testo, vediamo se viene aggiornato automaticamente. Piattaforma che consentirà di pubblicare blog orientati alla scrittura e all'insegna del minialismo, ne deriva una struttura molto semplice focalizzata sul contenuto. Non sono previsiti infatti, almeno in questa fase, commenti, integrazione con social network o altro.

Il database di fatto non esiste, è in sostanza costituito da file di testo memorizzati in una directory, che contengono ciascuno un articolo del blog, il testo può essere scritto anche utilizzando le regole del markdown.

I file vengono memorizzati all'interno della directory pubblica del proprio account di dropbox, si utilizza in questa fase una directory pubblica poiché non richiede autenticazione da parte dell'applicazione. In ogni caso, se non diversamente autorizzato, è solo l'utente ad avere accesso alla propria directory pubblica. Da valutare in seguito se integrare le API di dropbox per consentire all'utente di utilizzare una qualsiasi directory del proprio dropobox.

All'inizio del file di testo ci dovranno essere alcuni meta data che contengono il titolo dell'articolo, la data e l'ora, le categorie separate da virgola. I meta data per comodità sono preceduti da identificatori particolari, ipotizziamo di creare il file articolo1.txt.

TITLE: Titolo dell'articolo
DATE: gg/mm/aa hh:mm
CATEGORY: eventuale categoria
Da qui in poi il testo dell'articolo contenuto nel file articolo1.txt...

Lo script PHP, che per esemplificare chiameremo post.php dovrà così rilevare la presenza dei file contenuti nella directory e memorizzarne il contenuto per esempio in un array di variabili. Per ogni indice dell'array un file e quindi un articolo.

Nel caso del file articolo1.txt in quattro variabili dell'array sarà memorizzato il seguemte testo:

$title = Titolo dell'articolo

$date = gg/mm/aa hh:mm

$category = eventuale categoria

$content = Da qui in poi il testo dell'articolo contenuto nel file articolo1.txt

Attraverso un file di configurazione (che consentirà di specificare anche altre opzioni) si potrà decidere quanti articoli (indici dell'array) visualizzare. Si determinerà così il numero di articoli (partendo dal più recente) che l'utente visualizzerà accedendo alla pagina index.php. In pratica si otterrebbe una visualizzazione tipo questa.

Ciò che immagino è la possibilità di richiamare all'interno della pagina index.php lo script esterno post.php, che restituirà in sequenza il numero di articoli specificato nel file di configurazione. La pagina conterrà anche l'HTML e il CSS che servirà per realizzare il template del sito vero e proprio, ad esempio menu, barra laterale etc.

Template della pagina

Per realizzare il template dovrà essere possibile utlizzare liberamente le variabili che compongona la struttura dell'articolo, nel nostro esempio $title, $date, $category, $content.

Questo per poterle disporre a piacimento nel layout di pagina e poterle associare a un tag, utilizzabile poi nel foglio di stile CSS1 modificandone gli attributi grafici.

In pratica si deve poter realizzare un template tipo questo.

<div id = "titolo">`
    $title
</div>

<div id = "data">
    $date
</div>

<div id = "post">
    $content
</div>

Avendo specificato nel file di configurazione di visualizzare i tre articoli più recenti si avrà questa visualizzazione.

Identificando con un id specifico ciascun elemento sarà possibile controllarlo e formattarlo in maniera opportuna tramite il foglio di stile CSS. Volendo per esempio che il titolo del post sia in Arial, 24px, grassetto e centrato nel file CSS potremo specficare:

#titolo
{

    font: bold 24px Arial, Helvetica, sans-serif;

    text-align; center;

}

Così se volessimo visualizzare la data sopra il titolo non faremo altro che spostare il div id "data" sopra il div id "titolo". Questo metodo ci consente un totale controllo del layout della pagina.

Markdown

Come dicevo è possibile scrivere il testo dell'articolo utilizzando il markdown, un metodo di scrittura che consente di definire attraverso semplici identificatori, alcuni elementi che poi vengono trasformati in HTML quando la pagina viene renderizzata. Il markdown è stato inventato da John Gruber qui se ne posso trovare i dettagli. Ne è stata realizzata anche una evoluzione che si chiama mutimarkdown. Esiste poi un porting in PHP molto semplice da utilizzare che si può trovare qui. Molti editor di testo per desktop, iPad e iPhone supportano questo tipo di linguaggio e la sincronizzazione con Dropbox.

Volendo è anche possibile inserire direttamente del codice html nel testo. Questo è per esempio un video VIMEO inserito utilizzando html. Questa pagina è stata scritta anche con un iPad utilizzando l'applicazione Byword e il mutimarkdown. Il file index.php risiede sul server, va aleggere un file memorizzato nella directory pubblica del mio dropbox, ne interpreta il contenuto utilizzando la libreria markdown.php che genera l'HTML che a sua volta vine formattato da un foglio di stile CSS sempre memorizzato sul server.

Non mi dilungo sulla sintassi del markdown, se ne può trovare una descrizione dettagliata ai link di cui sopra.
